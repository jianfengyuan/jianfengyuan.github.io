Maven

Maven仓库的关系

  • 配置maven工程时, maven首先会寻找系统盘的本地仓库, 若本地仓库(b2b)没有相关jar包, 会联网到中央仓库寻找jar包, 中央仓库(central)放置了几乎所有的开源jar包. 公司开发中, 还会有自己的远程仓库保存自己的jar包, 开发项目时, 可以往公司的远程仓库(私服)中提取jar包, 也可以往远程仓库中上传jar包, 如果公司的远程仓库也没有相关jar包, 也会从中央仓库中下载.

  • 本地仓库的路径

    本地仓库的默认路径可以在maven根目录下的config文件夹中setting.xml中找到

Maven项目结构

scr/main/java目录 核心代码部分

scr/main/resources 配置文件部分

scr/test/java目录 测试代码部分

scr/test/resources 测试配置文件

scr/main/webapp 页面资源, js, css, 图片等资源

Maven常用命令

  • mvn clean: 删除target目录, 清除项目编译信息

  • mvn compile: 编译src中main的代码

  • mvn test: 编译src中test的代码

  • mvn package: 把项目进行打包, 把编译后的main和test代码放到target目录中, 可以在pom.xml配置文件中设置输出包的类型

  • mvn install: 在mvn package的基础上, 把包放到Maven的本地仓库中

maven生命周期

  • 默认生命周期: compile -> test -> package -> install -> deploy

  • 清理生命周期: clean

  • 每一个构建项目的命令都对应maven底层一个插件

Error

  • 自己写的Servlet不能被是被识别成Servlet

java.lang.ClassCastException: class com.kim.servlet.MyServlet cannot be cast to class javax.servlet.Servlet (com.kim.servlet.MyServlet is in unnamed module of loader org.apache.catalina.loader.WebappClassLoader @cc9ef8d; javax.servlet.Servlet is in unnamed module of loader org.codehaus.plexus.classworlds.realm.ClassRealm @6ec63f8)
at org.apache.catalina.core.StandardWrapper.loadServlet(StandardWrapper.java:1116)

原因: maven集成Tomcat插件, Tomcat插件中Servlet的包与Maven仓库中的包冲突

解决方法: 给Servlet相关依赖加上作用域

  • Tomcat启动异常

    org.eclipse.jdt.internal.compiler.classfmt.ClassFormatException

    原因: JDK和Tomcat版本不匹配(JDK版本过高/Tomcat版本过低)

    解决办法: 使用更高版本的Tomcat插件, 然后使用mvn tomcat7:run 启动服务器

  • 端口被占用
    找到哪个程序占用了该端口

    lsof -i tcp:8080

    杀死相关进程

    kill <pid>
